# Правила разметки корпуса

## Общие принципы

1. **Размечаются только персональные данные.** Обычные слова и нерелевантные фрагменты, по которым нельзя без дополнительной информации идентифицировать конкретного человека, не размечаются.
2. **Сущности не пересекаются:** один токен может принадлежать только одной сущности.
3. **Исходный текст не изменяется**, нормализация не выполняется.
4. **Каждая сущность размечается как единый непрерывный фрагмент.**

Инструмент разметки: [Doccano](https://github.com/doccano/doccano). 

## Классы сущностей

### PERSON — антропонимы

Размечаются имена, фамилии, отчества и их сочетания, если по ним можно идентифицировать конкретное физическое лицо.

Включаются:
- Полное ФИО: `Иван Иванович Петров`
- Фамилия + имя: `Петров Иван`, `Иван Петров`
- Инициалы + фамилия: `И.И. Петров`, `Петров И.И.`
- Имя с должностью: `директор Смирнов`, `гражданин Петров`
- Двойные фамилии, иностранные имена, сложные отчества

Не включаются:
- Имена исторических деятелей, публичных персон в нарративном контексте (если текст не о конкретном живом человеке)
- Антропонимы в составе названий улиц, организаций, объектов: `ул. Ленина`, `театр им. Чехова`
- Обращения без фамилии, если лицо не идентифицируемо: `уважаемый Иван` (без фамилии — не размечается, если контекст не позволяет установить личность)

### PHONE — номера телефонов

Размечаются все форматы записи телефонных номеров.

Включаются:
- С кодом страны и без: `+7 999 123-45-67`, `8 (495) 123 45 67`, `926-00-11`
- Со скобками, пробелами, дефисами, точками: `(926) 000.11.22`
- С текстовыми префиксами: `тел. 926-00-11`, `моб.: +7 999 000 00 00`
- С добавочными: `+7 495 123-45-67 доб. 101`
- Специфические форматы: `8-800-XXX-XX-XX`

Не включаются:
- Числовые последовательности, формально похожие на телефон, но являющиеся ID, артикулами, кодами по контексту

### EMAIL — адреса электронной почты

Размечается полный адрес вместе с доменной частью.

Включаются:
- Стандартный формат: `ivan.ivanov@mail.ru`
- С плюс-адресацией
- Разные регистры, нестандартные домены, различные символы в email

Не включаются:
- Технические URL, ссылки на ресурсы без `@`

### ADDRESS — адреса

Размечаются почтовые и фактические адреса, позволяющие установить место проживания или нахождения конкретного человека.

Включаются:
- Полные адреса
- С почтовым индексом
- Сокращённые варианты с маркерами адреса
- Адреса в различном порядке составляющих
- Адреса промышленных объектов, строительных площадок, если относятся к физическому лицу

Не включаются:
- Только регион, только город или только улица без дополнительного контекста, если не позволяют идентифицировать лицо
- Названия населённых пунктов в нарративном/новостном контексте без привязки к конкретному человеку

### ID — идентификаторы

Размечаются числовые и буквенно-цифровые идентификаторы, позволяющие установить личность.

Включаются:
- ИНН
- СНИЛС
- Номера документов, контрактов
- Номера договоров и заявок
- Банковские реквизиты и счета

Не включаются:
- Артикулы товаров, коды позиций, технические идентификаторы без привязки к физическому лицу

## Граничные случаи

| Ситуация | Решение |
|---|---|
| Несколько ФИО в одном предложении | Каждое ФИО — отдельная сущность |
| Адрес разбит по строкам | Если относится к одному месту — единая сущность |
| Сущности с текстовым префиксом | Префикс не включается в сущность |
