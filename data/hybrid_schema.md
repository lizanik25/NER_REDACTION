# Схема гибридного подхода

## Обоснование

Анализ трёх подходов к распознаванию персональных данных показал, что их эффективность зависит от типа сущности:

- **Rule-based (Yargy + regex)** обеспечивает практически идеальное качество для форматных сущностей (PHONE, EMAIL, ID), но уступает нейросети на контекстно-зависимых классах (PERSON, ADDRESS).
- **Slovnet (предобученный)** справляется с PERSON, но не обучен на PHONE/EMAIL/ID/ADDRESS.
- **Slovnet (дообученный)** даёт высокое качество по всем классам, но на сложных примерах ошибается там, где числовые последовательности формально похожи на PHONE или EMAIL.

Гибридная стратегия объединяет сильные стороны обоих методов и устраняет их слабые места.

## Архитектура

```
Входной текст
     │
     ▼
 Chunking
 (блоки ≤1500 символов, перекрытие 200 символов,
  разрезание по «мягким» границам: \n\n → \n → конец предложения)
     │
     ▼
HybridPIIExtractor
 ├── RuleBasedPIIExtractor ──────────────────────► PHONE, EMAIL, ID
 │    ├── ContextIdDetector
 │    ├── YargyPhoneDetector
 │    ├── YargyEmailDetector
 |    └── YargyIdDetector
 │
 └── SlovnetPIIExtractor ────────────────────────► PERSON, ADDRESS
      └── Slovnet NER (дообученная модель)
           └── Navec embeddings
     │
     ▼
 Разрешение пересечений
 (при конфликте приоритет у rule-based для PHONE/EMAIL/ID,
  у ml для PERSON/ADDRESS; при полном перекрытии - удаление дубликата)
     │
     ▼
 Пересчёт позиций в координаты исходного текста
     │
     ▼
 Фильтрация:
 ├── allowlist - исключить перечисленные фрагменты
 └── denylist  - принудительно пометить как ПД
     │
     ▼
TextAnonymizer
 ├── replace      → [PERSON], [PHONE], [EMAIL], [ADDRESS], [ID]
 ├── mask         → ****
 └── pseudonymize → случайный псевдоним того же класса
     │
     ▼
Анонимизированный текст + JSON-отчёт
```

## Параметры вызова

| Параметр | Тип | Описание | По умолчанию |
|---|---|---|---|
| `mode` | str | Режим анонимизации: `replace`, `mask`, `pseudonymize` | `replace` |
| `classes` | list[str] | Классы для обнаружения | все 5 классов |
| `threshold` | float | Порог уверенности предсказания модели (0.0-1.0) | `0.0` |
| `allowlist` | list[str] | Слова/фрагменты, не считающиеся ПД | `[]` |
| `denylist` | list[str] | Слова/фрагменты, принудительно помечаемые как ПД | `[]` |
